# Развёртывание бота-моста Telegram ↔ MAX

*Пошаговая инструкция. Составлена на основе реального опыта, с учётом всех подводных камней.*

---

## Важные нюансы — прочитайте до начала

### Про сервер: почему нельзя брать в России

Telegram заблокирован на территории России. Любой российский сервер — Timeweb, Selectel, VDSina, Aeza и другие — не сможет подключиться к Telegram. Бот просто не запустится, что бы вы ни делали.

Некоторые российские хостинги предлагают зарубежные IP или прокси до Telegram как платную опцию — но это заметно дороже обычного тарифа и менее надёжно.

**Вывод: нужен сервер за рубежом.**

### Про зарубежный сервер: сложности для россиян

Получить зарубежный хостинг из России сложно по двум причинам:

- Большинство европейских хостингов (Hetzner, DigitalOcean, Vultr) с 2022–2024 годов ограничили или полностью закрыли регистрацию для клиентов с российскими документами и адресами
- Российские карты (Мир, Visa и Mastercard выпущенные в РФ) не принимаются для оплаты на зарубежных хостингах

Hetzner, в частности, с января 2024 года не принимает клиентов с российскими адресами и удалил существующих. РКН также периодически блокирует сети Hetzner внутри России, хотя сам сервер при этом продолжает работать нормально.

### Варианты решения

**Вариант 1 (рекомендуемый): Российский хостинг с зарубежными серверами**

Некоторые российские хостинги продают серверы в зарубежных дата-центрах. Вы регистрируетесь на русском сайте, платите рублями с российской карты, но сервер физически стоит за границей — и Telegram API работает.

**Timeweb Cloud** (timeweb.cloud) — проверенный вариант:
- Есть серверы в **Казахстане** и **США** — оба региона работают с Telegram API
- Регистрация на русском, оплата рублями, российские карты принимаются
- При создании сервера выбирайте регион **Казахстан** или **США** (не Россию!)

**Вариант 2: Иностранный хостинг напрямую**

Если есть иностранный документ и карта иностранного банка — можно зарегистрироваться напрямую на Hetzner, DigitalOcean, Vultr и других. Серверы в Финляндии, Германии, Нидерландах — все работают с Telegram.

**Вариант 3: Посредники или знакомые за рубежом**

Российские компании-посредники арендуют серверы через свои зарубежные юрлица и предоставляют доступ за рубли. Дороже, но не требует иностранных документов. Или попросите знакомых за рубежом зарегистрировать аккаунт.

### Про MAX

MAX — российский мессенджер. Его серверы доступны с любого зарубежного хостинга без ограничений.

### Что нужно подготовить заранее

- Папка с файлами проекта на компьютере: `app/`, `Dockerfile`, `docker-compose.yml`, `requirements.txt`
- Файл `.env` с токенами (заполнить по шаблону `.env.example`)
- Токен Telegram-бота — получить у @BotFather в Telegram
- Токен бота MAX — получить в настройках MAX
- ID вашей Telegram-группы (отрицательное число) — узнать у @userinfobot
- ID топика в группе (если используете темы форума)
- ID группы в MAX (числовое)
- Ваш личный Telegram ID (узнать у @userinfobot) — для команды администратора `/status`
- Windows 10 или 11 на вашем компьютере

## Шаг 1. Получение доступа к серверу

Когда у вас есть доступ к зарубежному серверу, вы получаете три вещи:

- **IP-адрес** сервера (четыре числа через точку, например `12.34.56.78`)
- **Имя пользователя** — обычно `root`
- **Пароль**

⚠️ Сразу запишите все три в надёжное место. Потеря пароля означает потерю доступа к серверу.

Сервер должен работать на **Ubuntu 24.04**. Минимальные характеристики для бота: 1 CPU, 2 GB RAM, 20 GB SSD.

---

## Шаг 2. Подключение к серверу по SSH

SSH — способ управлять сервером через текстовые команды с вашего компьютера. Устанавливать ничего не нужно, встроен в Windows.

**Открыть терминал:**

Нажмите `Win + R`, введите `cmd`, нажмите Enter. Откроется чёрное окно — это терминал.

**Подключиться к серверу:**

```
ssh root@ВАШ_IP
```

- На вопрос `Are you sure you want to continue connecting?` — напишите `yes`, Enter
- На запрос `password:` — вставьте пароль правой кнопкой мыши. Символы не отображаются при вводе — это нормально. Нажмите Enter.
- Если сервер просит сменить пароль: введите старый (из письма), потом дважды новый придуманный.
- Когда появится строка `root@...#` — вы внутри сервера.

✅ Консоль можно закрыть в любой момент — бот на сервере продолжит работать. Чтобы зайти снова — откройте новый терминал и повторите команду `ssh root@ВАШ_IP`

## Шаг 3. Установка Docker

Docker устанавливается один раз. Вводите команды по одной, ждите завершения каждой.

**Команда 1 — обновить систему:**

```bash
apt update && apt upgrade -y
```

Займёт 1–3 минуты. В конце снова появится строка `root@...#`

**Команда 2 — установить Docker:**

```bash
curl -fsSL https://get.docker.com | sh
```

Займёт 2–5 минут. Если с первого раза выдала ошибку с сетью — введите повторно, обычно со второго раза проходит.

✅ Docker установлен если в конце вывода видите строку: `Docker version 29.x.x`

---

## Шаг 4. Копирование файлов на сервер

Используем `scp` — встроенный инструмент Windows для копирования файлов по SSH. Ничего устанавливать не нужно.

**Шаг 4.1 — Создайте папку на сервере:**

В терминале где открыт сервер введите:

```bash
mkdir -p /root/bridge
```

**Шаг 4.2 — Откройте второй терминал на компьютере:**

`Win + R` → `cmd` → Enter. Первый терминал с сервером не закрывайте.

**Шаг 4.3 — Скопируйте файлы (во втором терминале):**

Введите одну длинную команду целиком. Замените `ВАШЕ_ИМЯ` на имя пользователя вашего Windows и `ВАШ_IP` на IP сервера:

```
scp -r C:\Users\ВАШЕ_ИМЯ\Documents\bridge\app C:\Users\ВАШЕ_ИМЯ\Documents\bridge\Dockerfile C:\Users\ВАШЕ_ИМЯ\Documents\bridge\docker-compose.yml C:\Users\ВАШЕ_ИМЯ\Documents\bridge\requirements.txt root@ВАШ_IP:/root/bridge/
```

Введите пароль сервера. Появятся названия копируемых файлов — это нормально.

⚠️ Файл `.env` НЕ копируйте через scp — создайте его на сервере вручную в следующем шаге.

---

## Шаг 5. Создание файла с токенами (.env)

Создаём файл с секретными настройками прямо на сервере через встроенный редактор nano.

**В первом терминале (на сервере) введите:**

```bash
nano /root/bridge/.env
```

Откроется редактор. Вставьте содержимое файла правой кнопкой мыши. Шаблон:

```
TG_BOT_TOKEN=ваш_токен_telegram_бота
TG_GROUP_ID=id_вашей_tg_группы
TG_TOPIC_ID=id_топика_если_есть
MAX_BOT_TOKEN=ваш_токен_max_бота
MAX_GROUP_ID=id_вашей_max_группы
ADMIN_IDS=ваш_личный_telegram_id
REDIS_URL=redis://redis:6379
WEBHOOK_SECRET=любая_случайная_строка
```

⚠️ Строка `REDIS_URL` должна быть именно `redis://redis:6379` — не localhost. В Docker контейнеры общаются по имени сервиса, а не localhost.

Сохраните файл: нажмите `Ctrl + O`, затем `Enter`. Выйдите из редактора: `Ctrl + X`.

**Проверьте что файл сохранился:**

```bash
cat /root/bridge/.env
```

Должны появиться все строки с токенами.

## Шаг 6. Запуск бота

⚠️ Если бот запущен на вашем компьютере — остановите его перед запуском на сервере. Два одинаковых бота одновременно вызовут конфликт.

**Перейдите в папку проекта и запустите:**

```bash
cd /root/bridge
docker compose up -d --build
```

Первый запуск занимает 3–5 минут — Docker скачивает Python и Redis. Появится много текста. В конце снова строка `root@...#`

**Проверьте что контейнеры работают:**

```bash
docker compose ps
```

Должно показать два контейнера со статусом `Up`:

```
tg-max-bridge   Up
tg-max-redis    Up
```

**Проверьте логи:**

```bash
docker logs tg-max-bridge
```

Должны появиться строки про запуск моста. Если логи пустые — подождите 15 секунд и введите команду повторно.

## Шаг 7. Проверка работы

- Напишите сообщение в Telegram-группу — оно должно появиться в MAX-группе
- Напишите сообщение в MAX-группу — оно должно появиться в Telegram-группе
- Если оба направления работают — готово!

✅ Бот работает на сервере постоянно и самостоятельно. При перезагрузке сервера поднимается автоматически. Консоль можно закрывать.

## Управление ботом

Все команды вводить на сервере из папки `/root/bridge/`

Подключение к серверу: `ssh root@ВАШ_IP`

| Действие | Команда |
|----------|---------|
| Проверить статус | `docker compose ps` |
| Посмотреть логи | `docker logs tg-max-bridge` |
| Перезапустить бота | `docker compose restart bot` |
| Остановить всё | `docker compose down` |
| Запустить снова | `docker compose up -d` |
| Обновить код | `docker compose up -d --build` |

## Обновление кода бота

Когда получили новые файлы от разработчика:

**1. Скопируйте новые файлы (на компьютере, второй терминал):**

```
scp -r C:\Users\ВАШЕ_ИМЯ\Documents\bridge\app root@ВАШ_IP:/root/bridge/
```

**2. Пересоберите и перезапустите (на сервере):**

```bash
cd /root/bridge
docker compose up -d --build
```

Займёт 1–2 минуты. Данные Redis не потеряются.

## Типичные ошибки и решения

### Бот не подключается к Telegram

**Ошибка:** `Cannot connect to host api.telegram.org`

**Причина:** сервер в России — Telegram заблокирован.

**Решение:** перенести бота на зарубежный сервер (Timeweb Cloud — регион Казахстан или США).

### Неверный пароль при подключении

**Причина:** лишний символ или опечатка.

**Решение:** скопируйте пароль целиком, вставьте правой кнопкой мыши. Если несколько раз не получается — сбросьте пароль в панели управления хостингом.

### Логи бота пустые

**Причина:** Python буферизирует вывод по умолчанию.

**Решение:** добавьте переменную `PYTHONUNBUFFERED=1`. Введите на сервере:

```bash
sed -i 's/- REDIS_URL=redis:\/\/redis:6379/- REDIS_URL=redis:\/\/redis:6379\n      - PYTHONUNBUFFERED=1/' /root/bridge/docker-compose.yml
docker compose up -d --build
```

### Конфликт: два бота запущены одновременно

**Ошибка:** `TelegramConflictError: terminated by other getUpdates request`

**Причина:** бот запущен и на компьютере, и на сервере одновременно.

**Решение:** остановите бота на компьютере (`Ctrl+C` в том окне где он запущен).

### Контейнер постоянно перезапускается

**Решение:** смотрите логи — в конце будет конкретная ошибка:

```bash
docker logs tg-max-bridge
```

### Ошибка подключения к Redis

**Причина:** в `.env` указан `REDIS_URL=redis://localhost:6379`

**Решение:** исправить на `redis://redis:6379` и перезапустить бота.
